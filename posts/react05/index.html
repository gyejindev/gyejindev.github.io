React State와 Lifecycle

By Jogyejin
Posted 2024-05-05 8 min read

State개요

  • React에서의 State는 단언 뜻 그대로 상태를 뜻하며 Props처럼 JavaScript의 객체로 구성되어있다. 두 객체 모두 렌더링 결과물에 영향을 주는 정보를 갖고 있는데 중요한 차이가 있다 Props는 함수의 매개변수처럼 컴포넌트에 전달되는 반면에 state는 함수 내의 선언된 변수처럼 컴포넌트 안에서 관리된다는 점이다.

  • React는 지역변수, 전역변수의 값이 변경된다해서 컴포넌트를 다시 렌더링 하지 않는다 state를 사용하여 데이터를 관리하는 이유이다 (state 값이 변하면 관련 있는 컴포넌트들이 재렌더링되어 화면이 바뀐다)

  • 일반적으로 리액트에서는 유동적인 데이터는 변수에 담아서 사용하지 않고 useState()라는 리액트 함수를 사용하여 state라는 저장 공간에 담아 사용한다

  • State를 공부하기전에 중요한 부분이있는데 바로 Constructor 즉 생성자이다

    • 생성자는 다른 프로그래밍 언어에서도 사용되며 객체를 만드는 역할을 하는 함수이다 객체를 만들기위해 Class를 생성하며 Class안에 생성자는 해당 Class의 초기값을 설정해 주는데 사용된다

setstate는 뭘까

위에서 state는 어플리케이션 각 페이지(컴포넌트)에서 사용되는 데이터 즉 state데이터에 대해 알아보았는데,

해당 state의 값은 어떻게 바꾸는것일까?

1

this.state = {date: new Date()};

위처럼 this.state에 값을 바로 넣어주면 된다고 생각할 수 있지만 state에 직접 값을 넣는것은 해당 컴포넌트의 생성자에서만 가능하다!(state값을 직접 변경해도 render() 함수는 새로 호출되지 않는다)

그럼 생성자 이후에 state값을 바꾸기 위해서는 어떻게할까?

바로 아래에서 처럼 setstate함수를 사용하는것이다

setstate를 호출했을때에만 렌더링 함수가 호출되어 화면에 변경사항을 반영할 수 있다

1

this.setState({date: new Date()});

 class Clock extends React.Component {
   constructor(props) {
     super(props);
     this.state = {date: new Date()};
   }
 
   componentDidMount() {
     this.timerID = setInterval(
       () => this.tick(),
       1000
     );
   }
 
   componentWillUnmount() {
     clearInterval(this.timerID);
   }
 
   tick() {
     this.setState({
       date: new Date()
     });
   }
 
   render() {
     return (
       <div>
         <h1>Hello, world!</h1>
         <h2>It is {this.state.date.toLocaleTimeString()}.</h2>
       </div>
     );
   }
 }
 
 const root = ReactDOM.createRoot(document.getElementById('root'));
 root.render(<Clock />);

위 코드의 과정을 분석해보자

  • (1) root.render(<Clock />); 부분이 제일먼저 실행되어 React는 Clock 컴포넌트의 생성자(constructor)를 호출한다, 클래스 컴포넌트는 항상 props로 기본 constructor를 호출해야 한다 여기서 super키워드는 부모클래스 생성자의 참조 명령어이다 즉, super는 React.component의 생성자(constructor)를 가리킨다, 이작업을 하는 이유는 class Clock extends React.Component 에 의해서 Clock이라는 컴포넌트는 React.Component라는 부모 클래스에의해서 컴포넌트로 만들 수 있었는데 부모 클래스인 React.Component와 Clock 컴포넌트의 데이터 충돌(함수나 변수명이 같음)을 막기위해 상위 컴포넌트에서 전달해주는 속성값이다 해당 supuer(props)를한 이후에 this 접근이가능하다 이후 Clock이 현재 시각을 표시해야 하기 때문에 현재 시각이 포함된 객체로 this.state를 초기화한다

React에서 Component를 생성할 때 state 값을 초기화하거나 메서드를 바인딩할 때 construcotr()를 사용한다 React의 Component의 생성자는 해당 Component가 마운트 되기 전 호출되며 React.Component를 상속한 컴포넌트의 생성자를 구현할 때는 super(props)를 선언을 권고하고 있다. 이유는 this.props 사용 시 생성자 내에서 정의되지 않아 버그 발생 가능성이 생기기 때문이다.


  • (2) React는 Clock 컴포넌트의 render함수를 호출한다, 이를 통해 React는 화면에 표시되어야 할 내용을 구분하게되고 React는 Clock의 렌더링 출력값을 일치시키기 위해 DOM을 업데이트한다


  • (3) Clock 출력값이 DOM에 삽입되면, React는 componentDidMount() 생명주기 메서드를 호출한다 그 안에서 Clock 컴포넌트는 매초 컴포넌트의 tick() 메서드를 호출하기 위한 타이머를 설정하도록 브라우저에 요청한다


  • (4) 매초 브라우저가 tick() 메서드를 호출하며 그 안에서 Clock 컴포넌트는 setState()에 현재 시각을 포함하는 객체를 호출하면서 UI 업데이트를 진행한다 setState() 호출 덕분에 React는 state가 변경된 것을 인지하고 화면에 표시될 내용을 알아내기 위해 render() 메서드를 다시 호출한다 이 때 render() 메서드 안의 this.state.date가 달라지고 렌더링 출력값은 업데이트된 시각을 포함하며 React는 이에 따라 DOM을 업데이트한다.


  • (5) Clock 컴포넌트가 DOM으로부터 한 번이라도 삭제된 적이 있다면 React는 타이머를 멈추기 위해 componentWillUnmount() 생명주기 메서드를 호출한다
